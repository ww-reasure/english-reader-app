## 📋 设计方案

### 目标
实现三个体验优化功能：限时阅读、阅读统计、文章收藏

### 文件结构
```
js/
├── views/
│   ├── reading.js      ← 修改（限时模式 + 收藏按钮）
│   ├── history.js      ← 修改（收藏筛选）
│   ├── stats.js        ← 新增（统计页面）
│   └── chat.js         ← 修改（收藏状态同步）
├── db.js               ← 修改（articles 表加 favorite 字段 + readingStats 表）
├── router.js           ← 修改（添加 stats 路由）
└── helpers.js          ← 修改（添加 CountdownTimer 类）

css/
└── style.css           ← 修改（统计页面 + 限时模式样式）

index.html              ← 修改（添加 stats.js 脚本）
```

### 核心模块

#### 模块 1：限时阅读模式
- CountdownTimer 类（helpers.js）
- 阅读页面工具栏：开关 + 三档速度（150/200/250）
- 倒计时显示 + 进度条
- 超时提醒（不强制停止）

#### 模块 2：阅读进度追踪
- 新增 StatsView（views/stats.js）
- DB 查询方法：总阅读/连续天数/已学词/平均速度
- 四宫格统计卡片 + 阅读速度趋势（简化版）
- 导航栏添加「统计」入口

#### 模块 3：文章收藏/分类
- articles 表添加 favorite 字段（0/1）
- 阅读页面添加收藏按钮（⭐）
- 历史页面添加筛选：全部/收藏

### 限制条件
- 不破坏现有功能
- articles 表加字段需兼容旧数据（默认 favorite = 0）
- 统计页面数据从已有表查询，不新增表
- 限时模式可选，不强制

### 风险点
- DB 升级版本号需处理旧数据迁移
- 连续阅读天数计算需考虑时区
- 阅读速度趋势需要足够的数据量才有意义
